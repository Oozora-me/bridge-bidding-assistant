## 二、代码层面 Bug

### Bug 1: HCP_RULES_TEXT 标题重复

`buildSystemRulesText` 中拼接了 `【点力计算标准】\n`，而 `HCP_RULES_TEXT` 自身也以 `【点力计算标准 - 哈伦点力（HCP）】` 开头，导致 prompt 中出现两个标题：

```
【点力计算标准】
【点力计算标准 - 哈伦点力（HCP）】
- A = 4点，K = 3点...
```

**影响**: 可能干扰 AI 对 HCP 规则的理解。

### Bug 2: 体系 ID 不匹配

日志显示前端传的是 `nsSystem: natural`，但后端配置中定义的是 `natural-2over1`。

`getSystemDocContent('natural')` 找不到文档，回退到简略规则。但 `getEnabledConventions('natural')` 也可能找不到约定叫。

**影响**: 完整体系文档可能没有被注入到 prompt 中。

## 三、根因分析

### 3.1 模型能力不足（主因）

GLM-4-Flash 是轻量级模型，存在以下能力缺陷：
- **数值推理能力弱**: 无法正确执行 HCP 计算（A=4, K=3, Q=2, J=1 这种简单加法）
- **长上下文指令遵循差**: prompt_tokens 约 5000-10000，模型无法在大量文本中准确定位和执行规则
- **输出不稳定**: 同一手牌三次请求给出三个不同答案
- **幻觉严重**: 编造不存在的规则（如"T=1点"、"斯台曼在1♥后使用"）

### 3.2 代码 Bug（次因）

- 重复标题可能干扰 AI 理解
- 体系 ID 不匹配导致完整文档可能未注入
